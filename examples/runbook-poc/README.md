# Runbook POC Fixtures

These fixtures validate the canonical registry-driven Runbook A and Runbook B
workflow.

- Runbook A fixture proves a page brief can become a strict markdown outline
  through `RUNBOOK_A_PAGE_TO_MARKDOWN_OUTLINE.md`.
- Runbook B fixture proves a markdown outline can become a route-agnostic
  production page patch plan through
  `RUNBOOK_B_MARKDOWN_OUTLINE_TO_PRODUCTION_PAGE.md`.
- End-to-end fixture proves brief -> outline -> page package failures can be
  diagnosed without hidden human interpretation.
