# Generated Example: Blog/Document Body

Approval Status: Approved Example With Documented Stop Point

This package proves `templates/blog-document.md` can be filled and assembled
into a body-only `.tsx` educational document using approved section components
only. It also records the required stop point for true long-form article-body
needs.

## Files

- `filled-brief.md` - sample filled brief from the approved blog/document
  template.
- `BlogDocumentBodyExample.tsx` - body-only React composition using approved
  library imports only.
- `component-evidence.md` - registry ID, `specPath`, `libraryPath`, token, and
  screenshot evidence for every component used.
- `new-component-request.md` - documented request for a future approved
  article-body/content-block component.
- `final-checklist.md` - final pre-publish checklist for this example.

## Rules Proven

- Starts from `AI_START_HERE.md`, `READ_FIRST_AI.md`, `PREFLIGHT.md`, and
  `templates/blog-document.md`.
- Uses only component IDs from `registry/components.json`.
- Imports React implementations only from `library/src/components/` through the
  library index.
- Creates body content only; global chrome is excluded.
- Does not fake a long-form article layout with custom cards or CSS.
- Routes missing article-body needs to `templates/new-component-request.md`.
